Wireframe Notes

  • Screen 0a (Welcome): First impression with app branding and call-to-action. Simple, inviting design.
  • Screen 0b (Sign Up): Account creation with email/password or social sign-in options. Minimal friction.
  • Screen 0c (Partner Connection): Clear choice between inviting or joining a partner. Visual distinction between actions.
  • Screen 0d (Invite Partner): Generated 6-digit code displayed prominently with multiple sharing options. Shows waiting state.
  • Screen 0e (Join Partner): Simple code entry with large, easy-to-read input field. Validation hints provided.
  • Screen 0f (Connected): Celebration screen confirming successful pairing. Shows both user avatars.
  • Screen 1 (Home): Primary view showing current balance prominently. Quick access to recent expenses and settle up button.
  • Screen 2 (Add Expense): Streamlined form with large amount input, smart category selection, paid by selector, and simple split options. Designed for one-handed use.
  • Screen 2b (Custom Split): When "Custom" is selected, shows percentage inputs for each person with real-time calculation preview.
  • Screen 3 (Settle Up): Clear amount display with cash settlement option.
  • Screen 4 (Expense Details): Full expense breakdown showing who paid, how it was split, and the balance impact.
  • Screen 5 (Statistics): Monthly overview with total expenses and category breakdowns to help couples understand spending patterns.

Key UX Principles

  • ‚úÖ Speed: Add expense in 3 taps, no unnecessary fields
  • ‚úÖ Clarity: Balance always visible, plain language (no accounting jargon)
  • ‚úÖ Trust: Easy to edit/delete, transparent calculations
  • ‚úÖ Simplicity: Assumes 50/50 split, offers customization when needed
  • ‚úÖ Visual Hierarchy: Most important info (balance) is largest and most prominent
  • ‚úÖ Easy Pairing: Simple 6-digit code system, no complex setup
